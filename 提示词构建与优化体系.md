## 一、 核心架构：模块化提示词（Modular Prompting）
一个结构完整的高质量提示词，通常包含以下五个核心模块（你可以根据任务复杂度增减模块）：

1. **Role（角色设定）**：赋予模型特定的专业背景，激活其在该领域的专业词汇和深度思维。
2. **Context（背景信息）**：提供当前任务的上下文、你的当前水平或最终目的。
3. **Task（核心任务）**：明确、具体的行动指令。
4. **Constraints（约束条件）**：限制输出的范围、语气、避免使用的套话或特定的技术栈限制。
5. **Format（输出格式）**：规定输出的排版（如表格、Markdown、特定代码风格）。

#### 1. 科研文献阅读与分析 (Paper Reading)
在面对复杂的 AI 领域论文时，避免让模型“总结全文”，而是让它作为你的科研助手进行定向拆解。

- **框架示例**：**[Role]** 你是一位深耕人工智能领域的资深研究员，擅长快速剖析顶会论文的核心创新点。
**[Context]** 我正在进行关于大语言模型知识编辑（Knowledge Editing）的文献调研，需要评估这篇论文的方法是否适合我的研究。
**[Task]** 请阅读我提供的论文摘要和方法论部分，提取以下信息：1. 该方法解决的核心痛点是什么？2. 它与传统的全量微调（Fine-tuning）或常规的 RAG 方法相比，核心差异和优势在哪里？3. 实验部分证明了它在哪些指标上取得了 SOTA？
**[Constraints]** 保持学术严谨性，使用专业术语。如果论文中没有提及某点，请直接说明“未提及”，不要自行推断。
**[Format]** 请用 Markdown 格式输出，核心技术点请加粗，并用要点列表呈现。

#### 2. 代码审阅与生成 (Code Review & Generation)
代码任务需要极高的精确度，约束条件和背景信息尤为重要。

- **框架示例（审阅）**：**[Role]** 你是一位拥有多年经验的底层架构和高级 C++ 工程师。
**[Context]** 我正在编写一个并行计算的作业任务，使用了多线程处理数据，当前代码在运行大数据集时偶尔会出现崩溃。
**[Task]** 请审阅以下 C++ 代码，指出可能存在的内存泄漏、线程安全（如死锁或竞态条件）或性能瓶颈问题。
**[Constraints]** 不要重写整个文件，只需指出有问题的代码块，解释原因，并给出针对该代码块的修复建议。考虑缓存命中率和系统架构的底层优化。
**[Format]** 按照“问题位置 -> 问题原因 -> 修复代码片段”的结构输出。

#### 3. 课程概念解释 (Concept Explanation)
遇到晦涩的计算机科学或数学概念时，利用模型的类比能力和逐步拆解能力。

- **框架示例**：**[Role]** 你是一位擅长启发式教学的计算机科学教授。
**[Context]** 我正在学习 3D 计算机视觉，对 PointTransformer (PTv3) 中的某些注意力机制变体理解不够透彻。我有扎实的数据结构和常规 Transformer 基础。
**[Task]** 请向我解释该机制的运作原理。
**[Constraints]** 请使用我熟悉的二维图像处理或基础数据结构知识进行类比。包含具体的输入输出张量维度（Tensor shapes）变化过程。
**[Format]** 第一部分提供直观的类比，第二部分提供数学/逻辑层面的逐步拆解，最后给出一个简单的伪代码流程。

## 三、 提示词的动态优化与迭代逻辑
一个完善的提示词体系不是一成不变的，你需要建立一套**“试错 -> 修正 -> 沉淀”**的闭环机制。

1. **利用思维链 (Chain of Thought, CoT)**
当你发现模型在处理复杂逻辑（如算法分析或推导）时出现幻觉或跳跃，强制要求它展示思考过程。
  - *优化指令*：“在给出最终结论前，请一步步写出你的推理过程（Let's think step by step）。”
2. **少量样本提示 (Few-Shot Prompting)**如果模型无法准确掌握你想要的输出风格或格式，直接在提示词中提供 1-2 个成功的“输入-输出”示例。这比用长篇大论解释格式有效得多。
3. **建立反馈回路 (Feedback Loop)**
不要在一次不满意的回答后直接开启新对话。在同一对话中纠正它，这也是优化提示词的过程：
  - *“你刚才的回答太偏向应用层面了，请重新回答，这次深入到数学公式和算法复杂度的层面。”*
  - 当你通过多轮对话得到了完美的回答，问模型一个反向问题：*“如果我希望你下次直接生成这样高质量的回答，我应该给你提供怎样的初始提示词？”* 这样可以利用 LLM 帮你逆向工程出优秀的 Prompt。

不过，这里有一个纯前端的**核心技术难点**：出于安全机制，浏览器里的普通网页（`file://` 协议）是**无法直接静默覆盖保存本地文件**的。
为了实现你“直接打开网页查看，且修改能反馈到存储中”的需求，且不启动任何类似 Flask 的后端，你有两个非常优雅的纯前端替代方案：

## 方案 A：LocalStorage 实时缓存 + JSON 导入/导出（最稳定、最简单）
这是最经典的无后端做法。

- **读取与修改**：网页加载时，读取浏览器内置的 `localStorage`。你在网页上的任何拼接、修改、增删，都通过 JavaScript 实时 `JSON.stringify` 并保存到缓存中。刷新网页数据绝不会丢。
- **持久化存储**：在网页上放一个“导出为 JSON”的按钮。当你完成了一段时间的积累，点击它，浏览器会下载一个 `prompts.json` 文件到你的本地归档。下次换电脑时，用“导入 JSON”按钮把文件传进去即可恢复。

## 方案 B：使用 File System Access API（最符合你的终极需求）
现代浏览器（如 Chrome 或 Edge）提供了一个强大的 Web API。

- **工作原理**：你在网页上点击“打开文件”，选择你本地某个盘符下的 `prompt_system.json`。浏览器会请求一次读写权限。
- **无缝同步**：授权之后，你在网页上的所有修改，都可以通过 JS 直接调用 `fileHandle.createWritable()` 实时覆盖那个真实的本地 `.json` 文件！**完全不需要后端，实现了真正的本地文件双向绑定。**

## 具体方案设计：如何构建这个“提示词工作台”？
我建议你采用 **“纯 HTML + Tailwind CSS (通过 CDN) + 原生 JavaScript (或引入极简的 Vue)”** 的单文件结构，这样不需要配置任何 Node.js 或 Webpack 环境。

#### 1. JSON 数据结构设计
你的 JSON 应该像一个数据库，把提示词拆解成模块，方便你在前端自由拼接：

```json
[
  {
    "id": "research-01",
    "category": "论文阅读",
    "title": "3D视觉论文快速拆解",
    "modules": {
      "role": "你是一位顶级的 3D 计算机视觉与多模态（VLM）算法专家...",
      "context": "我正在调研低成本点云-文本对齐的方向...",
      "task": "请阅读以下摘要，提取其与常规 RAG 方法的核心差异...",
      "constraints": "保持学术严谨性，使用专业术语，按 Markdown 格式输出。"
    }
  }
]

```

#### 2. 前端界面布局（推荐三栏式）

- **左栏（侧边栏）**：树状列表。按类别（如课程学习、代码审阅、论文阅读）展示所有的提示词模板标题，点击即可选中。
- **中栏（编辑区）**：表单区域。将选中的提示词打散成多个文本框（Role, Context, Task 等）。你可以在这里直接修改文本，或者留出类似 `{{这里粘贴代码}}` 的占位符。
- **右栏（输出预览与操作）**：
  - 一个只读的文本框，用 JS 实时监听中栏的变化，将各个模块拼接成最终的完整 Prompt。
  - 一个硕大的 **“一键复制 (Copy to Clipboard)”** 按钮。
  - 底部的“保存到本地 JSON”按钮。

#### 3. 核心 JavaScript 逻辑（拼接逻辑）
你可以写一个简单的拼接函数，每次中栏的输入框发生 `input` 事件时触发：

```javascript
function generateFinalPrompt(modules) {
    let prompt = "";
    if(modules.role) prompt += `【角色设定】\n${modules.role}\n\n`;
    if(modules.context) prompt += `【背景信息】\n${modules.context}\n\n`;
    if(modules.task) prompt += `【核心任务】\n${modules.task}\n\n`;
    if(modules.constraints) prompt += `【约束与格式】\n${modules.constraints}\n\n`;
    // 可以添加动态变量替换逻辑
    return prompt;
}

```
